# Publication Checklist

- [ ] README explains what the project does and how to run it.
- [ ] Examples use fake data only.
- [ ] No credentials, personal data, connector exports, private notes, or raw logs.
- [ ] `.env` files are ignored; only `.env.example`, `.env.sample`, or `.env.template` may be committed.
- [ ] License and security note are present.
- [ ] Quality checks pass locally.
- [ ] Public-repo guard or an equivalent scan has been run before publishing.
